# 风险清单 (Risk Catalog)

> 本文档为 atlas 子技能的风险参考清单。
> 用于生成「边界条件与风险矩阵」时系统性识别代码中的潜在风险。

---

## 风险等级定义

| 等级 | 标识 | 判定标准 |
|------|------|----------|
| **高** | 🔴 HIGH | 可能导致崩溃 (crash)、安全问题 (security)、严重错误输出 (severe wrong output)、数据损坏 (data corruption) |
| **中** | 🟡 MED | 可能导致边界行为错误 (edge-case misbehavior)、控制不稳定 (control instability)、业务偏差 (business logic deviation) |
| **低** | 🟢 LOW | 可维护性 (maintainability)、命名歧义 (naming ambiguity)、magic number、文档缺失 (missing docs) |

---

## 1. 空值 (Null / Nil Dereference)

**等级**: 🔴 HIGH

### 检测提示

- 函数返回 Optional / nullable 类型但调用方未做 null check
- 集合取值（`map[key]`、`list[idx]`）后直接解引用
- 链式调用 `.foo().bar().baz()` 中间环节可能返回 null
- 第三方库 API 文档标注 nullable 但代码未防御

### 常见场景

- 解析 JSON/配置文件时字段缺失
- 缓存未命中返回 nil 后直接使用
- 工厂方法返回 nil 表示创建失败，但调用方未处理
- 回调/闭包捕获外部变量在异步上下文中变为 nil

### 建议测试

- 输入为 null / nil 时函数行为
- 关键字段缺失时的降级路径
- Optional 链与强制解包的边界情况

---

## 2. 越界 (Out of Bounds)

**等级**: 🔴 HIGH

### 检测提示

- 循环索引使用 `<` vs `<=` 边界
- 数组/切片访问无长度检查
- `list.get(index)` 无边界保护
- 字符串 substring 起止位置计算
- 递归终止条件依赖于数组长度但可能 off-by-one

### 常见场景

- 空数组/列表上的遍历或取值
- 环形缓冲区索引回绕计算错误
- 分页查询最后一页数量不足
- 多维数组维度不匹配

### 建议测试

- 空集合、单元素集合
- 索引 = 长度（刚好越界）
- 索引 = 长度 - 1（最后一个元素）
- 负索引（语言支持时）

---

## 3. 除零 (Divide by Zero)

**等级**: 🔴 HIGH

### 检测提示

- 除法运算 `/` 的分母为变量或表达式
- 取模 `%` 的分母同理
- 平均值计算 `sum / count`，count 可能为 0
- 比例/归一化计算中分母来自减法结果（可能为零）
- 向量归一化时向量长度为零

### 常见场景

- 统计指标计算（空数据集）
- 归一化/标准化（全零向量）
- 速率/频率计算（时间间隔为零）
- 百分比/占比计算（总量为零）

### 建议测试

- 分母为 0 的显式测试
- 分母为极小值（浮点精度）
- 空数据集上的聚合计算
- 全零输入的归一化

---

## 4. 溢出 (Overflow)

**等级**: 🟡 MED

### 检测提示

- 整数运算（特别是乘法、幂运算）可能超出类型范围
- 累加器/计数器在大数据量下增长
- 类型转换（如 `int64` → `int32`、`float` → `int`）
- 位运算移位超过类型位宽
- 阶乘、排列组合等指数增长计算

### 常见场景

- 时间戳计算（毫秒→秒→分→时累积）
- 大循环中的计数器
- 哈希值计算中的乘法累积
- 嵌入式系统中固定位宽的传感器值

### 建议测试

- 输入为类型最大/最小值
- 连续累加到溢出
- 大数乘法
- 有符号/无符号混用

---

## 5. NaN / Inf (Special Float Values)

**等级**: 🟡 MED

### 检测提示

- `0.0 / 0.0` → NaN
- `1.0 / 0.0` → +Inf
- `sqrt(负数)` → NaN
- `log(0)` → -Inf, `log(负数)` → NaN
- NaN 参与比较运算结果恒为 false
- NaN 传播：任何包含 NaN 的算术运算结果为 NaN

### 常见场景

- 科学计算/数值优化中的除法
- 机器学习 loss 计算中的 log(probability)
- PID 控制器中误差为零时的积分项
- 三角函数在特殊角度的 tan 值

### 建议测试

- 产生 NaN 的输入组合
- 产生 Inf 的输入组合
- NaN 作为后续计算输入的传播行为
- `isfinite()` / `isnan()` 检查是否存在

---

## 6. 未初始化状态 (Uninitialized State)

**等级**: 🔴 HIGH

### 检测提示

- 变量声明但未赋值即使用
- 类/结构体字段缺少默认值
- Optional 类型未显式初始化
- 静态/全局变量依赖初始化顺序
- 配置项未设置时的默认行为

### 常见场景

- 对象构造后直接使用但缺少 init 步骤
- 状态机初始状态未定义
- 延迟初始化（lazy init）在首次访问前被使用
- 条件分支跳过了初始化但后续代码使用变量

### 建议测试

- 对象创建后不调用 init 直接使用
- 状态机从非法状态开始
- 配置文件缺失关键字段时的启动行为

---

## 7. 状态未重置 (State Not Reset)

**等级**: 🟡 MED

### 检测提示

- 循环/迭代中累积状态未在每次迭代开始时清零
- 单例/静态变量在多次使用间残留旧值
- 连接/会话复用时前次请求的数据泄漏
- 测试夹具 (test fixture) 未在测试间清理

### 常见场景

- 累加器变量在批处理间未清零
- 请求上下文 (request context) 在异步处理中残留
- 缓存过期策略未触发导致返回过期数据
- 状态机处理完一个事务后未回到初始状态

### 建议测试

- 连续两次调用同一函数，验证第二次不携带第一次的状态
- 批处理场景下各批次间的独立性
- 测试用例执行顺序敏感（说明有状态残留）

---

## 8. 状态被错误重置 (State Incorrectly Reset)

**等级**: 🟡 MED

### 检测提示

- reset/clear 操作覆盖了不应清除的数据
- 异常处理路径中过早重置状态
- 多线程/协程中一个路径重置了另一个路径正在使用的状态
- 部分重置：只清了部分字段但遗漏了其他关联字段

### 常见场景

- 重连逻辑中清空了认证状态导致无限重连
- 心跳超时后重置连接但丢失了未确认的消息
- 错误恢复逻辑中重置了用户配置
- 多任务调度中任务切换时上下文保存/恢复不完整

### 建议测试

- 异常发生后状态是否保持一致
- 重置操作后系统是否能正常重新初始化
- 并发场景下重置与读取的竞态

---

## 9. 并发问题 (Concurrency Issues)

**等级**: 🔴 HIGH

### 检测提示

- 共享可变状态无锁保护
- check-then-act 模式非原子操作
- 双重检查锁定 (double-checked locking) 实现错误
- 闭包/lambda 捕获循环变量
- 死锁：锁获取顺序不一致
- 活锁：重试逻辑无退避

### 常见场景

- 多线程读写同一集合/Map
- 异步回调中修改共享状态
- 单例模式在并发环境下的多次初始化
- 数据库事务的隔离级别不足导致幻读/不可重复读
- 信号量/条件变量的 spurious wakeup

### 建议测试

- 并发压力测试（多线程/协程同时操作）
- 锁获取顺序互换场景
- 长时间持有锁的场景（超时机制）
- race condition 注入测试

---

## 10. 资源泄漏 (Resource Leaks)

**等级**: 🟡 MED

### 检测提示

- 打开文件/连接但无 `close()` / `with` / `using` 保护
- 事件监听器注册但从未注销
- 临时文件创建但未清理
- 内存分配（特别是循环中）无释放路径
- 线程/协程创建但无 join / cancel 机制

### 常见场景

- 异常路径跳过资源清理
- 长时间运行的服务中累积泄漏
- 连接池耗尽（连接未归还）
- 大对象缓存在全局 Map 中永不释放

### 建议测试

- 反复执行后内存/文件描述符是否增长
- 异常中断后的资源清理验证
- 长时间运行的压力测试

---

## 11. 错误处理缺失 (Missing Error Handling)

**等级**: 🔴 HIGH

### 检测提示

- 函数返回 error / Result 类型但调用方忽略
- `try-catch` 块为空或只做日志记录
- panic / throw 后无恢复机制
- 错误信息丢失（被覆盖或吞掉）
- 超时未设置导致永久阻塞

### 常见场景

- I/O 操作（网络、文件、数据库）未处理失败
- 类型转换/解析未处理格式错误
- 外部服务调用无重试/降级
- 断言 (assert) 在生产代码中用于非不可恢复的条件

### 建议测试

- 模拟每个外部依赖失败
- 输入格式错误的场景
- 错误传播链是否完整（上层能否感知底层错误）
- 超时场景

---

## 12. Magic Number

**等级**: 🟢 LOW

### 检测提示

- 代码中出现未解释的数字常量
- 阈值、超时、重试次数等硬编码
- 同一数字在多处出现但含义不同
- 与业务规则相关的数字未定义为常量

### 常见场景

- 算法中的系数/权重
- 超时值（ms/s 混用）
- 缓冲区大小
- 重试次数/间隔

### 建议测试

- 修改该数字值时需要改动多少处（散弹式修改 smell）
- 值为 0 或负数时的行为
- 不同环境下该值是否应该不同

---

## 13. 单位不一致 (Unit Inconsistency)

**等级**: 🔴 HIGH

### 检测提示

- 时间单位混用（秒 vs 毫秒 vs 微秒）
- 角度单位混用（度 vs 弧度）
- 距离/长度单位不一致（米 vs 厘米 vs 英尺）
- 字节单位（B vs KB vs KiB vs MB）
- 货币单位（元 vs 分 vs cents）
- 温度单位（℃ vs ℉ vs K）

### 常见场景

- 不同 API 使用不同单位但未在接口文档明确
- 配置文件使用人类可读单位但代码使用 SI 单位
- 硬件接口（传感器输出 vs 代码使用的单位）
- 跨系统集成时各方单位约定不同

### 建议测试

- 边界值在单位转换后的精度
- 极大/极小值在转换后是否溢出
- 类型标注了单位但实际传入其他单位值

---

## 14. 输出范围不匹配 (Output Range Mismatch)

**等级**: 🟡 MED

### 检测提示

- 函数返回值范围与调用方预期不一致
- 归一化输出范围不明确（`[0,1]` vs `[-1,1]` vs `[0,255]`）
- 角度输出度 vs 弧度
- 概率值超出 `[0,1]`
- 颜色值范围（`0-255` vs `0.0-1.0`）
- 控制信号范围与执行器限制不匹配

### 常见场景

- ML 模型输出需要 clip 但未做
- 传感器数据范围超出 ADC 量程
- 控制算法输出超出执行器可接受范围
- 坐标系转换后范围不一致

### 建议测试

- 输出值的上下界测试
- 输出值是否超出接收方可接受范围
- 边界值（0、最大值、最小值）的正确性

---

## 15. 外部依赖失败 (External Dependency Failure)

**等级**: 🔴 HIGH

### 检测提示

- 网络请求无超时设置
- 数据库查询无连接池管理
- 文件系统操作假设文件一定存在
- 第三方 API 假设一定返回成功
- 环境变量/配置项假设一定存在
- DNS 解析假设一定成功

### 常见场景

- 微服务调用链中某个节点不可用
- 数据库连接中断或超时
- 配置中心不可达
- 证书过期 / token 过期
- 依赖服务的 API 版本变更

### 建议测试

- 每个外部依赖的超时/失败场景
- 网络分区的行为
- 依赖服务返回异常状态码
- 重试/熔断/降级机制是否生效

---

## 16. 硬件安全风险 (Hardware Safety Risks)

**等级**: 🔴 HIGH

### 检测提示

- 控制指令无范围限制 (clamping / limiting)
- 急停 (emergency stop) 逻辑缺失或不完整
- 看门狗 (watchdog) 未启用或超时设置过长
- 安全关键参数无冗余校验
- 传感器故障检测缺失
- 执行器反馈未监测
- 上电/断电时序未考虑

### 常见场景

- 机器人关节角度/速度/力矩超限
- 电机驱动器接收非法 PWM 值
- 温度控制无上限保护
- 电池管理系统中电压/电流超限
- 运动控制中轨迹规划超出工作空间
- 通信中断时执行器的默认行为（应进入安全状态）

### 建议测试

- 控制指令超出物理限制时的行为
- 急停信号在各种运行状态下的响应
- 传感器故障/断连时的降级行为
- 通信中断后的超时保护
- 上电/断电时的安全状态
- 看门狗超时触发的恢复行为

## 附录：风险识别流程

```
1. 识别代码中的数据流和控制流
2. 对每个关键操作节点，逐项检查上述 16 类风险
3. 根据风险等级定义标注 HIGH / MED / LOW
4. 为每个识别出的风险设计对应测试用例
5. 在风险矩阵中汇总：风险 → 等级 → 触发条件 → 建议测试 → 缓解措施
```
