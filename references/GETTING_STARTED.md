# Harness 零基础入门指南

> 下载一个 App，说一句话，你的软件就做好了。

---

## 三步上手

```
1. 打开 OpenCode    2. 输入 /harness     3. 说你想做什么        4. 看着它做出来
   ┌──────────┐      ┌──────────┐        ┌──────────────┐      ┌──────────────┐
   │  OpenCode │─►   │ /harness │  ───►  │ "做个记账App" │─►   │  你的软件 ❤️  │
   └──────────┘      └──────────┘        └──────────────┘      └──────────────┘
```

就这三步。下面展开说说每一步怎么做。

---

## 1. 打开 OpenCode

### 1.1 安装

去 [opencode.ai](https://opencode.ai) 下载桌面客户端。macOS / Windows / Linux 都有。

双击安装，打开。跟装微信一样。

> 如果你更喜欢用终端，也可以命令行安装：
>
> ```bash
> curl -fsSL https://opencode.ai/install | bash
> ```

### 1.2 连接 AI

第一次打开 OpenCode 时，它需要一个「AI 大脑」来工作。

**推荐选择**：

| AI 提供商                | 特点                   | 适合               |
| ------------------------ | ---------------------- | ------------------ |
| **DeepSeek V4** ⭐ | 编程能力强、性价比极高 | **新手首选** |
| 智谱 GLM                 | 中文友好，便宜         | 中文项目           |
| Claude                   | 编程能力强             | 预算充足时         |
| GitHub Copilot           | GitHub 登录即用        | 已有 Copilot 账号  |

**怎么连**：在 OpenCode 里输入 `/connect`，选你要用的提供商，按提示注册登录。几分钟搞定。

> 推荐选 **DeepSeek V4**。编程能力第一梯队，价格只有 Claude 的几分之一。去 [platform.deepseek.com](https://platform.deepseek.com) 注册拿 API Key 就行。

### 1.3 启用 Harness

OpenCode 本身是个「裸」的 AI 程序员——聪明但没有纪律。Harness 让它变成一个**靠谱的** AI 程序员。

在 OpenCode 的输入框里输入斜杠命令：

```
/harness
```

这会加载 Harness 工程纪律系统。加载完成后，你的 AI 程序员就变成了一个有流程、有检查、有保障的工程团队。

> 💡 **每次新开 OpenCode 项目时**，输入一次 `/harness` 就行。之后的对话里 Harness 会自动工作，你不需要再管它。

### 1.4 打开你的项目文件夹

在 OpenCode 里打开你想放代码的文件夹（或者新建一个空文件夹）。

OpenCode 会自动分析项目结构，生成一份 `AGENTS.md` 帮它理解你的项目。

**到此为止，你的准备工作就结束了。** 没有别的要装，没有配置文件要改。

---

## 2. 告诉它你想做什么

### 2.1 你需要会的唯一技能

**用自然语言清楚描述你想要什么。**

就这么简单。你扮演产品经理，AI 扮演程序员。

**❌ 模糊的说法**：

> 做个网站

**✅ 清晰的说法**：

> 我想做一个个人记账的网页应用。用户可以记录每天的花费和收入，按类别看统计图表。数据保存在浏览器本地就行，不需要注册登录。界面要简洁好看。

### 2.2 两种模式：先想再做

OpenCode 有两个模式，按 **Tab 键**切换：

| 模式               | 图标 | 作用                                        |
| ------------------ | ---- | ------------------------------------------- |
| **计划模式** | 📋   | AI 只提方案，不改动代码。适合想清楚再动手。 |
| **构建模式** | 🔨   | AI 直接写代码、改文件。适合明确了就直接干。 |

**推荐流程**：先在计划模式里聊清楚，再切到构建模式让它实现。

### 2.3 怎么描述需求

一个好模板：

```
我想做 [什么]
给 [谁] 用的
需要能 [功能1]、[功能2]、[功能3]
不需要 [明确不做的]
参考 [你见过的类似产品]
```

举个例子：

> 我想做一个团队周报工具。
> 给 5-10 人的创业团队用。
> 每个人填写本周完成了什么、下周计划、需要帮助的事项。组长可以汇总查看。
> 不需要复杂的审批流程，越简单越好。
> 参考 Notion 的数据库视图。

### 2.4 跟 AI 聊天就是你的全部工作

发完需求之后，AI 可能会追问一些细节：

> 🤖 **AI**：这个周报工具，每个人填写的频率是每周一次吗？
> 🤖 **AI**：汇总视图需要导出功能吗（比如导出 PDF）？
> 🤖 **AI**：需要支持图片上传吗？

你只需要像聊天一样回答。不确定的就说「你帮我推荐」。

---

## 3. 看着它做出来

确认需求后，切到构建模式，说一句「开始做吧」。AI 就会：

```
📝 写需求文档 → 📐 规划方案 → ✂️ 拆分任务 → ⚡ 写代码 → ✅ 测试验收
```

全程自动。你喝杯咖啡就行。

### 3.1 中间会发生什么

AI 会在界面里展示它正在做的事情——创建文件、写代码、跑测试。你能实时看到。

偶尔它会停下来问你：

- 「这个功能涉及用户认证，属于高风险操作，你确认要这样设计吗？」
- 「我发现现有代码有一个冲突，你希望怎么处理？」

回答完它继续干。

### 3.2 做完了怎么看

AI 完成后会告诉你做了什么。你可以：

- 让 AI 打开应用看看效果
- 在浏览器里预览网页项目
- 让 AI 帮你检查做得对不对

### 3.3 不满意怎么办

直接说哪里不满意：

> 「统计图表的饼图颜色太少了，加更多颜色区分」

AI 会自动补上。

### 3.4 做坏了怎么办

OpenCode 有 `/undo` 命令——一键撤销上一步修改。

就像 Ctrl+Z 撤销打字一样，只不过撤销的是整个代码修改。

## Harness 是什么？

上面的流程里，你没看到「Harness」这个词——但它一直在后台工作。

### 一句话解释

**Harness 是 AI 程序员的工作纪律系统。**

打个比方：

- AI = 一个超级聪明但容易走神的程序员
- Harness = 一个严格的项目经理，盯住 AI 每一步都做对

没有 Harness，AI 可能会跳过测试、忘掉边界情况、写完才发现方向错了。

有 Harness，AI 必须：先想清楚 → 按计划做 → 做完检查 → 出报告。

### Harness 自动保护你

你不需要手动触发 Harness。它会根据你的修改自动判断风险：

| 风险级别 | 什么时候                 | 发生什么                      |
| -------- | ------------------------ | ----------------------------- |
| 🟢 低    | 改文档、修错别字、加测试 | AI 自己搞定                   |
| 🟡 中    | 做新功能                 | AI 走完整流程，自动推进       |
| 🔴 高    | 改用户认证、核心逻辑     | **AI 停下来问你**       |
| ⛔ 极高  | 改数据库、部署配置、密钥 | **AI 必须你批准才能动** |

> 小事 AI 自己处理。大事 AI 来找你确认。你不需要盯着每一步。

### 怎么让 Harness 为你工作

你只需要**像平时一样说话**。Harness 会自动识别你在做什么：

| 你说的话                | Harness 自动做什么                   |
| ----------------------- | ------------------------------------ |
| 「我有个想法，想做 XX」 | 追问细节 → 写需求 → 做计划 → 实现 |
| 「帮我做这个功能」      | 走规格→计划→任务→实现全流程       |
| 「这里有个 bug」        | 定位 → 修复 → 验证                 |
| 「做得怎么样了」        | 生成评估报告                         |
| 「项目结构有点乱」      | 分析架构，给优化建议                 |

## 实战：做一个待办事项应用

走一遍完整流程。

### 打开 OpenCode，开始对话

在 OpenCode 的输入框里打字：

> /harness 我想做一个待办事项网页应用。
> 用户可以添加待办、标记完成、删除。
> 数据保存在浏览器本地，不需要登录。
> 界面要好看，手机和电脑都能用。

### AI 追问，你回答

```
🤖 AI：需要给待办设置截止日期吗？
你：加上吧，截止日期快到的时候标红。

🤖 AI：需要分类功能吗？比如「工作」「生活」？
你：加上分类，用颜色区分。

🤖 AI：好的，我整理一下需求...
```

### 切到计划模式，审阅方案

按 **Tab** 切到计划模式。AI 会展示它的实现方案：

```
技术方案：
- 纯前端应用（HTML + CSS + JavaScript）
- 数据存储：浏览器 localStorage
- 界面：响应式布局，适配手机和电脑

文件结构：
- index.html — 主页面
- style.css — 样式
- app.js — 核心逻辑
```

看一眼，觉得 OK 就切到构建模式。

### 切到构建模式，开干

按 **Tab** 切到构建模式，说：

> 方案没问题，开始做吧

AI 就会自动创建文件、写代码、跑测试。你等着就行。

### 完成后查看

AI 做完后，告诉它：

> 打开浏览器让我看看效果

你的待办事项应用就出现在浏览器里了。

### 想加新功能？

继续在 OpenCode 里说：

> 加一个功能：待办可以设置优先级（高/中/低），高的排最前面

AI 自动走流程加上去。

## 常见问题

### 我完全不懂编程，能用吗？

**能。** 你的角色是「产品经理」——提需求、做决策、看结果。AI 负责所有技术细节。

### 我需要装什么？

只需要装 OpenCode。其他的（Python、Git、依赖库……）AI 会帮你装。

### AI 写的代码靠谱吗？

Harness 帮你把关：

- 每个功能都有**自动化测试**验证
- 有**需求文档**对照检查
- 高风险操作**必须你亲自批准**

### 不满意怎么办？

直接告诉 AI 哪里不满意。或者用 `/undo` 撤销上一步。

### 出了问题怎么办？

告诉 AI：「出问题了，[描述你看到的错误]。」大部分问题 AI 可以自己修复。

如果修不好，在 OpenCode 里输入 `/undo` 回到上一个能用的状态。

### OpenCode 和 Cursor 有什么区别？

| 特点   | OpenCode                     | Cursor     |
| ------ | ---------------------------- | ---------- |
| 界面   | 桌面 App + 终端 + Web        | 代码编辑器 |
| 优势   | 开源、可装插件（如 Harness） | 所见即所得 |
| 装插件 | ✅ 支持                      | ✅ 支持    |

两个可以一起用。OpenCode 做架构和自动化，Cursor 做细节微调。

遇到不认识的词，在这里查。
