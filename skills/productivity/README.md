# Productivity

Process and workflow skills for governed engineering.

- **[harness-grill](./harness-grill/SKILL.md)** — Stress-test a plan or spec by asking pointed questions one at a time. Probes assumptions, edge cases, and implicit requirements. Use before spec finalization or when requirements feel fuzzy.
- **[harness-architecture-review](./harness-architecture-review/SKILL.md)** — Find architecture improvement opportunities: shallow modules, DDD dependency violations, testability gaps. Use periodically to prevent AI-accelerated code rot.
